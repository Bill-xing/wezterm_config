## 2. tmux

### 2.1 当前环境里的关键设定

- 前缀键仍是 `Ctrl-b`
- 开启鼠标：可点击 pane、切换 pane、拖动边界、滚轮滚动
- 复制模式使用 `vi` 风格
- 窗口编号从 `1` 开始，pane 编号也从 `1` 开始
- pane 导航统一成 `i / k / j / l = 上 / 下 / 左 / 右`
- 借鉴 theniceboy 工作流，新增 session / window 快捷键与更强状态栏
- 启用了 `tmux-yank`、`tmux-resurrect`、`tmux-continuum`

### 2.2 最常用快捷键

| 快捷键 | 作用 | 来源 |
| --- | --- | --- |
| `Ctrl-b` | 进入 tmux 前缀模式 | 本机配置 |
| `Ctrl-b C-c` | 新建或切换 session | 本机配置 |
| `Ctrl-b c` | 新建 window，继承当前目录 | 本机配置 |
| `Ctrl-b |` | 左右分屏 | 本机配置 |
| `Ctrl-b -` | 上下分屏 | 本机配置 |
| `Ctrl-b i/k/j/l` | 切换到上/下/左/右 pane | 本机配置 |
| `Ctrl-b I/K/J/L` | 向上/下/左/右调整 pane 大小，每次 5 格 | 本机配置 |
| `Ctrl-b r` | 重新加载 `~/.tmux.conf` | 本机配置 |
| `Alt-o` | 新建 window | 本机配置 |
| `Alt-1` 到 `Alt-0` | 直达第 1 到第 10 个 window | 本机配置 |
| `Ctrl-1` 到 `Ctrl-0` | 直达第 1 到第 10 个 session；不存在会自动创建 | 本机配置 |
| `Ctrl-b 1` 到 `Ctrl-b 0` | 把当前 window 移到目标 session 并切过去 | 本机配置 |
| `Alt-f` | 当前 pane 最大化 / 还原 | 本机配置 |

### 2.3 关闭 / 删除 pane、window、session

| 快捷键 / 命令 | 作用 | 备注 |
| --- | --- | --- |
| `Ctrl-b x` | 关闭当前 pane | tmux 当前环境默认键位；会弹确认 |
| `exit` / `Ctrl-d` | 退出当前 shell；通常也会顺手关掉当前 pane | 当前 pane 里只跑这个 shell 时最直接 |
| `Ctrl-b &` | 关闭当前 window | tmux 当前环境默认键位；会弹确认 |
| `Ctrl-b :kill-session` | 删除当前 session | 先按 `Ctrl-b` 再按 `:` 输入命令并回车 |
| `tmux kill-session -t <session名>` | 删除指定 session | 例如：`tmux kill-session -t 3` |
| `tmux ls` | 列出当前所有 session | 删除前先看名字最稳妥 |

### 2.4 复制 / 滚动 / 选中

| 快捷键 | 作用 | 备注 |
| --- | --- | --- |
| `Ctrl-b [` | 进入复制模式 | 默认 tmux 行为，当前为 `vi` 风格 |
| `v` | 在复制模式里开始选择 | 本机配置 |
| `y` | 在复制模式里复制并退出 | 本机配置 |
| `i/k/j/l` | 复制模式里按上/下/左/右移动 | 本机配置 |
| `I/K` | 复制模式里快速上/下移动 5 行 | 本机配置 |
| `J/L` | 复制模式里跳到行首 / 行尾 | 本机配置 |
| 鼠标滚轮向上 | 先进入复制模式，再逐行向上滚动 | 本机配置，滚动被刻意调慢 |
| 鼠标滚轮向下 | 在复制模式里逐行向下滚动，滚到底会退出 | 本机配置 |

### 2.5 会非常有用的插件快捷键

| 快捷键 | 作用 | 来源 |
| --- | --- | --- |
| `Ctrl-b Ctrl-s` | 保存当前 tmux 会话 | `tmux-resurrect` 默认键位 |
| `Ctrl-b Ctrl-r` | 恢复上次保存的 tmux 会话 | `tmux-resurrect` 默认键位 |

### 2.6 使用建议

- 如果你长期开多个项目，最值钱的是：`Ctrl-b C-c`、`Ctrl-1..0`、`Ctrl-b 1..0`、`Ctrl-b i/k/j/l`、`Alt-o`、`Alt-f`
- 需要“删东西”时可以这样记：
  - 删 pane：`Ctrl-b x`
  - 删 window：`Ctrl-b &`
  - 删 session：`Ctrl-b :kill-session` 或 `tmux kill-session -t <session名>`
- 当前配置已经把 pane、window、session 三层操作统一到一套更强的工作流里，日常几乎不用再碰 tmux 默认那批零散命令

### 3.3 autosuggestion 的体感快捷键

当前启用了 `zsh-autosuggestions`，虽然你没有手工再绑专门的接受键，但插件默认会在这些动作里“接收建议”：

| 动作 | 常见按键 | 效果 |
| --- | --- | --- |
| 向右移动一个字符 | `Right` | 接受整条灰色建议 |
| 移动到行尾 | `End` | 接受整条灰色建议 |
| 向前跳一个单词 | `Ctrl-Right` | 接受建议中的下一个单词 |

也就是说，在你这套环境里：

- 看到灰色建议后，直接按 `Right` 或 `End`，通常就能整条补全
- 想只吃掉一个词，直接按 `Ctrl-Right`

## 4. WezTerm

### 4.1 当前环境里的自定义行为

- `Cmd-Enter`：切换全屏
- `Cmd-d`：横向分屏
- `Cmd-Shift-d`：纵向分屏
- `Cmd-w`：关闭当前 pane，不是关闭整个 tab
- `Cmd + 单击链接`：打开链接
- 右键抬起：如果有选区，则直接复制选区到系统剪贴板

### 4.2 当前环境最值得记的快捷键

| 快捷键 | 作用 | 来源 |
| --- | --- | --- |
| `Cmd-Enter` | 全屏切换 | 本机配置 |
| `Cmd-d` | 左右分 pane | 本机配置 |
| `Cmd-Shift-d` | 上下分 pane | 本机配置 |
| `Cmd-w` | 关闭当前 pane | 本机配置，覆盖了默认“关闭 tab” |
| `Cmd-t` | 新建 tab | WezTerm 生效中的默认键位 |
| `Cmd-1` 到 `Cmd-9` | 切换到指定 tab | WezTerm 生效中的默认键位 |
| `Cmd-{` / `Cmd-}` | 前一个 / 后一个 tab | WezTerm 生效中的默认键位 |
| `Cmd-c` / `Cmd-v` | 复制 / 粘贴 | WezTerm 生效中的默认键位 |
| `Cmd-f` | 搜索当前滚动缓冲区 | WezTerm 生效中的默认键位 |

### 4.3 pane 与复制模式

| 快捷键 | 作用 | 来源 |
| --- | --- | --- |
| `Ctrl-Shift-Left/Right/Up/Down` | 在 pane 间移动焦点 | WezTerm 生效中的默认键位 |

| `Ctrl-Shift-x` | 进入复制模式 | WezTerm 生效中的默认键位 |

进入复制模式后，当前环境基本就是一套非常像 Vim 的操作：

| 快捷键 | 作用 |

### 4.4 使用建议

- 如果你把 tmux 跑在 WezTerm 里，建议分清两层：
  - 终端分屏：`Cmd-d` / `Cmd-Shift-d`
  - tmux 分屏：`Ctrl-b |` / `Ctrl-b -`
- 如果你已经在 tmux 里长期工作，WezTerm 更适合负责：
  - tab 管理
  - 系统剪贴板
  - 搜索历史输出
  - 临时性的 GUI 级 pane 分割

## 5. LazyVim / Neovim

### 5.1 当前环境的真实情况

- 当前 `~/.config/nvim/lua/config/keymaps.lua` 只额外改了两件事：
  - `j`：按“屏幕行”向下移动
  - `k`：按“屏幕行”向上移动

- 当前 `<leader>` 是 `Space`

| 快捷键 | 作用 | 来源 |

## 7. yazi

### 7.1 当前环境的真实情况

- 当前同时存在：
  - `~/.config/yazi/yazi.toml`
  - `~/.config/yazi/keymap.toml`
- 说明你现在已经不是 yazi 默认键位，而是一套自定义文件管理工作流
- 当前自定义项主要包括：
  - manager 布局、排序、标题格式
  - opener / open rules
  - 图片、PDF、压缩包、代码等预览器
  - 与 tmux 对齐的 `ikjl` 导航
- 当前预览参数仍然保留：
  - `max_width = 1200`
  - `max_height = 1600`
  - `image_quality = 70`
  - `image_delay = 40`
  - `image_filter = "triangle"`

### 7.2 最常用键位

| 快捷键 | 作用 | 备注 |
| --- | --- | --- |
| `q` | 退出 yazi | 官方 quick start 明确说明 |
| `Q` | 退出但不改变 shell 当前目录 | 常和 `y` wrapper 搭配 |
| `F1` / `~` | 打开帮助 | 官方 quick start 明确说明 |
| `i` / `k` / `j` / `l` | 上 / 下 / 左 / 右导航 | 本机自定义，和 tmux 对齐 |
| `Left/Down/Up/Right` | 同上 | 箭头键也可用 |
| `gg` | 到顶部 | 很高频 |
| `G` | 到底部 | 很高频 |
| `I` / `K` | 快速上 / 下移动 5 行 | 本机自定义 |
| `J` / `L` | 后退 / 前进目录历史 | 本机自定义 |
| `Ctrl-p` | 通过 fzf 跳转到目录或定位文件 | 本机自定义 |
| `z o` | 通过 zoxide 跳转目录 | 本机自定义 |
| `g Space` | 通过交互提示跳目录 | 本机自定义 |

### 7.3 文件操作里最值得记的一组

| 快捷键 | 作用 | 备注 |
| --- | --- | --- |
| `Space` | 选中 / 取消选中当前文件 | 文件批量操作核心键 |
| `v` | 选中全部 / 进入选择语义 | 高频操作 |
| `Enter` / `l` | 打开文件或进入目录 | 最常用 |
| `r` | 交互式打开选中文件 | 本机自定义 |
| `yy` | 复制文件 | 常见默认键位 |
| `dd` | 剪切文件 | 常见默认键位 |
| `pp` | 粘贴文件 | 常见默认键位 |
| `dD` | 永久删除 | 本机自定义 |
| `T` / `M` | 新建文件 / 目录 | 本机自定义 |
| `cw` / `a` / `A` | 重命名 | 本机自定义 |
| `F` / `f` / `/` | 内容搜索 / 过滤 / 查找 | 在文件很多时很好用 |

说明：

- yazi 的帮助页非常值得多用一次，因为它会直接显示当前版本、当前层级的实际键位
- 如果你之后加了 `keymap.toml`，这部分建议重新生成一版

### 7.4 使用建议

- 如果你只想先把 yazi 用顺手，优先记：
  - `i k j l`
  - `gg`
  - `G`
  - `Space`
  - `Enter`
  - `yy dd pp`
  - `F`
  - `f`
  - `Ctrl-p`
  - `F1`
